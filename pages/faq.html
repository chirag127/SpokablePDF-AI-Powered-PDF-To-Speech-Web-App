Frequently Asked Questions

Find answers to common questions about Readable Spokable PDF

Readable Spokable PDF is a browser-based tool that converts technical PDF documents into natural, spoken-friendly content optimized for text-to-speech (TTS) systems. It uses AI to transform code blocks, tables, mathematical notation, and other technical elements into descriptive, easy-to-listen-to text.

No installation required! This is a pure web application that runs entirely in your browser. Just open the website and start using it. You will need a Google AI Studio API key, which you can get for free.

The application works with PDFs that contain selectable text. This includes most digitally-created PDFs like technical documentation, research papers, textbooks, and reports. Scanned PDFs (images of text) are not supported as they require OCR, which is not included in this tool.

  1. Visit Google AI Studio
  2. Sign in with your Google account
  3. Click "Get API Key" or "Create API Key"
  4. Copy the generated key (starts with "AIza")
  5. Paste it in the Settings page of this application

The free tier includes 15 requests per minute and 1,500 requests per day, which is suitable for most use cases.

Yes! Your API key is stored only in your browser's LocalStorage and is never transmitted to any server except Google's Gemini API endpoints (when making transformation requests). The application runs entirely in your browser, so your API key never leaves your device otherwise.

You can clear your API key at any time using the "Clear API Keys" button in Settings, or use the "Erase All Data" option to remove everything.

The application itself is free and open source. However, using the Google Gemini API may incur costs depending on your usage:

  • Free Tier: 15 requests/minute, 1,500 requests/day (suitable for occasional use)
  • Paid Tier: Higher limits with pay-per-use pricing

See Google AI Pricing for current rates.

All your data is stored locally in your browser:

  • Settings & API keys: LocalStorage
  • PDF files, batches, logs: IndexedDB

Nothing is stored on external servers. Your data stays on your device unless you explicitly export it.

Only the text content from your PDF and images (if multimodal processing is enabled) are sent to Google's Gemini API for transformation. This is necessary for the AI to convert technical content to natural speech.

Your API key is included in request headers but is managed by Google's secure infrastructure. The application does not send data to any other third parties.

The application can be loaded offline after the first visit (with appropriate caching), but you need an internet connection to make API calls to Google Gemini for the actual transformation. PDF extraction and generation can work offline, but the AI transformation requires internet access.

Processing time depends on several factors:

  • Document size: Larger documents take longer
  • Batch size: Smaller batches mean more API calls
  • API rate limits: Free tier has limits that slow processing
  • Network speed: Affects API call latency

Typical processing: ~5-10 seconds per batch. A 100-page document might take 5-15 minutes depending on settings.

Turbo Mode enables parallel processing of multiple batches simultaneously, which can significantly speed up transformation. However, it increases the risk of hitting API rate limits. It's recommended only for users with paid API accounts or when processing small documents.

You can configure the number of parallel batches in Settings (default: 3).

Common reasons for processing failure:

  • Invalid API key: Check Settings and test your key
  • Rate limits (429 errors): Increase Retry Delay, decrease Parallel Chunks, or add a backup API key
  • Model not available: Try a different model or check Diagnostics for available models
  • Network issues: Check your internet connection
  • API timeout: Increase the timeout in Settings

Use the Diagnostics page to export logs for troubleshooting.

Rate limit errors occur when you make too many requests too quickly. Try these solutions:

  1. Increase Retry Delay: In Settings, increase the delay between retries (e.g., 3000-5000ms)
  2. Disable Turbo Mode: Turn off parallel processing
  3. Increase Rate Limit Delay: Add more delay between requests (e.g., 2000ms)
  4. Add Backup API Key: Use a second API key for automatic failover
  5. Upgrade API Plan: Consider upgrading to a paid tier for higher limits

Try these approaches:

  • Edit Prompts: Customize the transformation prompts in Settings to better suit your document type
  • Adjust Temperature: Lower temperature (0.5-0.7) for more focused output, higher (1.2-1.5) for more creative
  • Try Different Models: Some models are better at certain tasks. Try gemini-3-pro-preview for complex documents
  • Reduce Batch Size: Smaller batches give the AI more context per chunk
  • Increase Overlap: More overlap between batches improves continuity

Yes! Click the Cancel button during processing. You can also download partial results if some batches have already been processed successfully.

When "Include Images" is enabled in Settings, the application:

  1. Extracts images from the PDF
  2. Converts them to base64 format
  3. Sends them to Gemini's multimodal endpoints along with nearby text
  4. Receives detailed descriptions of the figures
  5. Includes those descriptions in the output PDF

Note: If the selected model doesn't support images, the app automatically falls back to text-only processing.

Tokens are the basic units that AI models use to process text. The application uses a rough estimate of ~4 characters per token to split your document into appropriately-sized batches.

You can adjust the Batch Size in Settings. The default is 10,000 tokens (roughly 40,000 characters or about 7,000 words).

Visit the Diagnostics page and click "Export Diagnostic Bundle". This creates a ZIP file containing:

  • Application logs
  • Session information
  • Settings (with API keys redacted)
  • System information

This bundle is useful for troubleshooting or when requesting support.

Currently, the application processes one PDF at a time. However, you can queue multiple processing jobs by opening multiple browser tabs. Each tab maintains its own state independently.

There's no hard file size limit in the application, but practical limits include:

  • Browser memory: Very large PDFs (>100MB) may cause performance issues
  • API limits: Larger documents require more API calls and may take longer
  • IndexedDB quota: Browser storage is typically limited to a few hundred MB

For best results, documents under 50MB are recommended.
